ملف الإعدادات وبيانات الوصول لوسائل التواصل (بالعربي)

الغرض
- هذا الملف (`config.json`) اختياري ومصمم لتسهيل وضع بيانات الوصول (مثل توكن صفحة فيسبوك أو توكن بوت تليجرام) على جانب العميل أثناء مرحلة الإختبار أو التطوير.
- عند وضعه في نفس مجلد `index.html`، سيحاول التطبيق قراءة القيم ونسخها إلى `localStorage` حتى تكون جاهزة في واجهة لوحة الأدمن.

تحذير أمني مهم
- حفظ مفاتيح الوصول (Access Tokens) أو توكنات البوت في ملف `config.json` ظاهر أو في `localStorage` داخل المتصفح غير آمن للإنتاج.
- للأغراض الإنتاجية يجب استخدام مخزن سري على الخادم (Server-side) أو خدمة serverless لحفظ هذه المفاتيح وتنفيذ عمليات النشر عبر السيرفر بدلاً من المتصفح.

المفاتيح المدعومة (مثال `config.json`)
```json
{
  "fb_page_id": "YOUR_FACEBOOK_PAGE_ID",
  "fb_page_token": "YOUR_FACEBOOK_PAGE_ACCESS_TOKEN",
  "ig_token": "YOUR_INSTAGRAM_ACCESS_TOKEN_OR_PAGE_TOKEN",
  "tg_bot_token": "YOUR_TELEGRAM_BOT_TOKEN",
  "tg_chat_id": "YOUR_TELEGRAM_CHAT_ID",
  "deploy_token": "OPTIONAL_DEPLOY_TOKEN"
}
```

أين تضع الملف
- ضع `config.json` في جذر المشروع (نفس مجلد `index.html` و`app.js`).
- عند فتح الصفحة، سيحاول التطبيق جلب `config.json` ونسخ القيم المتوفرة إلى `localStorage` إذا لم تكن موجودة مسبقاً (حتى لا يتم استبدال القيم التي أدخلها الأدمن يدوياً).

كيف يستخدم التطبيق هذه القيم
- `fb_page_id` و `fb_page_token`: تُستخدم لمحاولة النشر على صفحة فيسبوك عبر مساعد النشر في الواجهة (Client-side attempt).
- `ig_token`: يمكن استخدامه كتوكن انستجرام أو كتوكن صفحة فيسبوك/انستجرام عند التوفر.
- `tg_bot_token` و `tg_chat_id`: تُستخدم من قبل مساعد التليجرام لإرسال صورة أو رسالة باستخدام البوت الخاص بك.
- `deploy_token`: قيمة اختيارية يستخدمها زر "حفظ إلى الخادم" إذا ربطت نقطة نهاية (endpoint) للخادم.

اختبار سريع (مقترح)
1. أنشئ `config.json` في مجلد المشروع بجانب `index.html` واملأ الحقول بالقيم الحقيقية أو بقيم تجريبية.
2. شغّل سيرفر محلي بسيط لخدمة الملفات (مستحسن بدل فتح الملف مباشرة). مثال باستخدام PowerShell وPython:

```powershell
# من مجلد المشروع
python -m http.server 5500; # سيفتح السيرفر على http://localhost:5500
```

3. افتح المتصفح واذهب إلى `http://localhost:5500` ثم افتح لوحة الأدمن. القيم الموجودة في `config.json` ستُنسخ إلى `localStorage` إذا لم تكن موجودة مسبقاً.
4. يمكن التحقق من القيم عبر أدوات المطور → Application → Local Storage وستجد المفاتيح: `fb_page_id`, `fb_page_token`, `ig_token`, `tg_bot_token`, `tg_chat_id`.

ملاحظات وخيارات لاحقة
- إن أردت تجنب تخزين أي مفاتيح على جانب العميل، اترك `config.json` خارج المشروع واستخدم واجهة الأدمن للصق المفاتيح مؤقتاً للاختبار.
- أستطيع أيضاً إضافة زر "استيراد ملف إعدادات" في لوحة الأدمن لرفع `config.json` عبر واجهة بدلاً من استضافته.
- إذا تريد، أستطيع تهيئة مثال لخدمة serverless (Node.js / AWS Lambda) لتخزين المفاتيح بشكل آمن وتنفيذ النشر عن طريق الخادم.

إذا تحب أجهز لك نسخة مُشفّرة محلياً أو أضبط زر استيراد في لوحة الأدمن، أخبرني أي طريقة تفضّلها.
